# 页面空白问题调试指南

## 快速检查步骤

### 1. 清除浏览器缓存
- **Chrome/Edge**: `Ctrl + Shift + Delete` → 清除缓存
- 或使用**无痕模式**访问：`Ctrl + Shift + N`
- 或**硬刷新**：`Ctrl + F5` 或 `Ctrl + Shift + R`

### 2. 检查浏览器控制台（最重要！）

按 `F12` 打开开发者工具，查看：

#### Console 标签页
查看是否有红色错误信息，常见错误：
- `Failed to load module script`
- `Uncaught ReferenceError`
- `Cannot read property`
- `404 Not Found`

#### Network 标签页
检查以下文件是否成功加载：
- ✅ `index.html` - 状态码应该是 200
- ✅ `assets/index-xxx.js` - 状态码应该是 200
- ✅ `assets/index-xxx.css` - 状态码应该是 200

如果看到 404 错误，说明资源路径有问题。

### 3. 检查页面源代码

右键页面 → "查看页面源代码"，确认：
- `<script>` 标签中的路径是否正确
- 路径应该是：`/assets/index-xxx.js`（不是 `/src/main.js`）

### 4. 检查 Vue 应用是否挂载

在浏览器控制台输入：
```javascript
document.getElementById('app')
```

如果返回 `null`，说明 `#app` 元素不存在。
如果返回元素但没有内容，说明 Vue 应用没有正确挂载。

## 常见问题及解决方案

### 问题1：资源路径 404

**症状**：Network 标签页显示 JS/CSS 文件 404

**原因**：`vite.config.js` 的 `base` 配置与部署路径不一致

**解决**：
1. 确认部署路径是 `/`
2. 确认 `vite.config.js` 中 `base: '/'`
3. 重新构建和部署

### 问题2：JavaScript 执行错误

**症状**：Console 有红色错误

**可能原因**：
- Vue 版本不兼容
- 模块加载失败
- 语法错误

**解决**：
1. 检查错误信息
2. 确认所有依赖已正确安装
3. 检查代码是否有语法错误

### 问题3：页面完全空白（无错误）

**症状**：页面空白，但控制台没有错误

**可能原因**：
- CSS 加载失败，内容存在但不可见
- Vue 应用挂载失败
- 字体加载失败

**解决**：
1. 检查 Network 标签页，确认 CSS 文件加载成功
2. 在控制台输入：`document.querySelector('.app-container')`
   - 如果返回元素，说明 Vue 已挂载，可能是 CSS 问题
   - 如果返回 `null`，说明 Vue 未挂载

### 问题4：字体加载失败

**症状**：页面显示但样式异常

**解决**：
- 检查 Google Fonts 是否可访问
- 如果无法访问，可能需要使用本地字体或 CDN

## 测试步骤

1. **打开开发者工具**（F12）
2. **切换到 Network 标签页**
3. **刷新页面**（F5）
4. **检查所有请求**：
   - 所有文件状态码应该是 200
   - 如果有 404，记录是哪个文件
5. **切换到 Console 标签页**：
   - 查看是否有错误
   - 如果有错误，复制错误信息

## 如果仍然无法解决

请提供以下信息：
1. 浏览器控制台的完整错误信息（Console 标签页）
2. Network 标签页的截图（显示哪些文件加载失败）
3. 页面源代码（右键 → 查看页面源代码）
